How the pre-flight check works

Heat from use is normal and recoverable. Before every run, the gate reads all seven joints, scores each against its own limit, and acts on the weakest: collect when it is cool, rest the actuator to cool first once it reaches its data-at-risk line (so the run is recorded clean), or reject a run that reads too hot to trust. The score, verdict, and breakdown are stamped onto every episode. Below is the gate on the hardest run it still collected; click any run for its breakdown.

Predictive maintenance, per joint

At day's end, the same sensors feed a drift detector for each joint, on two channels: temperature and motor current. The trend watches the daily mean, not the peak: a faster-heating motor runs hotter on average even with resting, so the mean carries the drift the per-run readings hide. A joint is flagged only when it both drifts statistically and degrades meaningfully. Watch where maintenance was called (blue), and where it reached quarantine, no longer serviceable (red).
